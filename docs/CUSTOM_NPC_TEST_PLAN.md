# Custom NPC Test Plan

Status: automated audit/repair tooling added; final visual check still needs an in-game pass.

Run this in a creative test world with commands enabled:

`/function ascendant_identity:npc_test/kit`

The function gives the CustomNPCs authoring tools, profile-key name tags, and sample Ascendant gear for fast NPC setup.

The live function intentionally uses command-safe CustomNPCs tools plus vanilla placeholder gear. This avoids one bad modded item id breaking the entire function at load time. After the NPC editor opens, equip the prettier modded showcase gear manually from JEI so the exact live item id is confirmed before saving a clone.

Root-cause note: the previous kit used `irons_spellbooks:truthseeker`; the current client rejected that id in command parsing, so Minecraft discarded the whole function and reported it as unknown.

NPC Wand use note: right-clicking air opens the remote editor list. That list is blank until NPCs already exist nearby or in the world. To create the first test NPC, right-click a block with the NPC Wand, then edit the newly spawned NPC.

Dynamic identity note: the name tags now use setup keys such as `ar:guild_clerk`, `ar:rank_examiner`, `ar:guard_captain`, and `ar:mira_ash`. They are not final labels. The visible rank/name/level/role line is generated by `customnpcs/scripts/ecmascript/ascendant_npc_identity.js`.

- Current safe baseline: the script reads the profile key from the NPC's temporary name and writes styled always-visible name text.
- Root-cause note: the CustomNPCs title slot only appears at close range and renders with angle brackets, so the fallback keeps level and role in the main name line.
- Existing-NPC repair note: CustomNPCs embeds a full copy of the script inside each NPC entity. Updating `customnpcs/scripts/ecmascript/ascendant_npc_identity.js` does not automatically update already placed NPCs. Run `python scripts\customnpcs-identity-audit.py` to inspect the active world and `python scripts\customnpcs-identity-audit.py --apply` after closing Minecraft to repair saved NPC entities. The old Rank Examiner prototype stored `ar_rank = guild_staff` and later `Unranked`, both of which are stale for the B-Rank Rank Examiner profile.
- Next step after this baseline passes: add stored `ar_level`, `ar_rank`, and `ar_role` values or move the system into a tiny custom helper mod.

## Automated Checks

Run this from the pack root:

`powershell -ExecutionPolicy Bypass -File scripts\check-custom-hero-system.ps1 -SyncClient`

When Minecraft is closed and saved NPCs need repair, run:

`powershell -ExecutionPolicy Bypass -File scripts\check-custom-hero-system.ps1 -SyncClient -Apply`

This runs:

- `node scripts\test-customnpcs-identity.js`
- `python scripts\customnpcs-identity-audit.py`
- `python scripts\check-pack.py`

The audit script scans saved CustomNPC entities, compares their embedded script copy against the repo script, verifies visible name/title/stored rank data, and can repair mismatches when Minecraft is closed.

Open the NPC Scripter after setting the profile key, then paste the contents of `customnpcs/scripts/ecmascript/ascendant_npc_identity.js` into the large black code box. The script rewrites the display name from the profile key.

- Rank Examiner: apply `ar:rank_examiner`; script should render `[B-Rank] Rank Examiner | Lv.52 Examiner`.

## Acceptance Checks

- NPC Wand opens and saves changes.
- Right-clicking a block with the NPC Wand creates the first NPC.
- Right-clicking air may open the remote editor list; a blank list is normal before NPCs exist.
- Profile-key name tags or temporary profile names are converted into styled display text by the script.
- The NPC Scripter black code box contains the Ascendant identity script and scripting remains enabled.
- The visible level comes from the profile baseline script, not from manually typed final name text.
- NPCs can visibly hold modded weapons and wear modded armor.
- Styled names remain readable above the NPC.
- Friendly NPCs do not attack players.
- Guard/rival combat works in a controlled creative test against vanilla mobs.
- Mob Cloner can save and respawn a finished prototype.
- Save/reload and disconnect/rejoin do not break the NPCs.
- `/function ascendant_identity:npc_test/kit` appears after `/reload` and grants the kit without an unknown-function error.
- `/function ascendant_identity:npc_test/fix_rank_examiner` resets the nearest old Rank Examiner prototype to the plain `ar:rank_examiner` profile key and B-Rank stored data. The offline audit/repair script is still the authoritative fix for stale embedded NPC scripts.

If this feels good but the nameplates still look too flat, the next step is the small Ascendant Nameplates client overlay instead of trying to force a server/plugin nameplate system into Forge.
