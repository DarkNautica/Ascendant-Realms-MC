# Ascendant Codex — grimoire art & previews

Previews and notes for the in-game **Ascendant Codex** (a Patchouli book), styled as an
Arcane Void grimoire to match the skill tree / HUD.

## Where the real files live (not here)
Patchouli 1.20+ requires the **resource-pack book layout** (`use_resource_pack: true`):
the `book.json` lives in a datapack, but the **content lives in assets**.
- **book.json** (data side): `config/openloader/data/ascendant_realms_codex/data/ascendant_realms/patchouli_books/ascendant_codex/book.json`
  (mirrored to `datapacks/` and `openloader/`). Must keep `use_resource_pack: true`.
- **Categories / entries** (assets side, always-on via KubeJS):
  `kubejs/assets/ascendant_realms/patchouli_books/ascendant_codex/en_us/{categories,entries}/`.
  (The old copies under the datapack `.../en_us/` are ignored now and can be left as-is.)
- **Grimoire textures** (assets, always-on via KubeJS): `kubejs/assets/ascendant_realms/`
  - `textures/gui/codex_book.png` — the 512x256 book GUI (Patchouli layout preserved).
  - `textures/item/ascendant_codex.png` + `models/item/ascendant_codex.json` — held book.
  - `textures/gui/codex/banner_getting_started.png` — chapter banner.

## Styling (book.json)
Dark void pages, luminous text. Colors: text `D7E3FF`, headers `9DE7FF`, links/keys `FFD98A`,
progress bar `7CF2FF`. Macros for consistent inline styling:
`$(rune)` cyan-bold keywords, `$(gold)` keybinds, `$(em)` violet-italic, `$(warn)` orange,
`$(soft)` muted asides. Use `$()` to reset.

## HOW IT ACTUALLY LOADS (important)
This Patchouli build (1.20.1-85) only discovers `book.json` inside **mod jars**
(`data/<modid>/patchouli_books/...`) and in the **external folder**
`<gamedir>/patchouli_books/<book>/`. It does NOT discover books from OpenLoader/world
datapacks, so the datapack copy never registered ("Invalid book").

The Codex therefore ships as an **external colocated book**:
- `patchouli_books/ascendant_codex/book.json` (`use_resource_pack: false`)
- `patchouli_books/ascendant_codex/en_us/{categories,entries}/` (content colocated here)
- Textures still come from the KubeJS assets pack (`ascendant_realms:...`).

Because the external loader registers under the `patchouli` namespace, the book id is
**`patchouli:ascendant_codex`**. Get it in-game with:
`/give @p patchouli:guide_book{"patchouli:book":"patchouli:ascendant_codex"}`
The `kubejs/client_scripts/ascendant_jei_aliases.js` tooltip/JEI alias points at this id.

(For distribution to other installs, copy the `patchouli_books/ascendant_codex/` folder into
each instance's game directory, or bundle it inside a mod jar under that mod's namespace.)


## Full first edition (complete)
The Codex is now fully written: **11 chapters, 40 entries**, with item/mob/image visuals
(`patchouli:spotlight`, `patchouli:entity`, `patchouli:image`) drawn from the pack's real
content and verified IDs (Bounty Board, dragons, unique weapons, spellbooks, backpacks,
bosses, artifacts). Content is generated by `scripts/generate-ascendant-codex-content.py`
into the external folder `patchouli_books/ascendant_codex/en_us/` (repo + instance).

Chapters: Getting Started · The Hunter's Path · The Guild & Bounties · Combat · Magic & Mana
· The Ascendant Web · Gear, Loot & Rarity · The World · Gates & Rifts · Threats & Bosses ·

Swords' Runic Grimoire, the Bestiary).
